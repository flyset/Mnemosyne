# PORE Methodology

Problem-Oriented Requirements Engineering anchors requirements in observable user or stakeholder problems.

## Rule

No problem, no requirement. Every objective, scope item, and acceptance criterion must trace to a stated problem.

## Problem Format

```text
P<n>: As a [role], I experience [problem], because [underlying reason or constraint].
```

## Derivation

For each problem, identify:

- root causes and constraints;
- an observable success condition;
- scope boundaries; and
- acceptance evidence that proves resolution.
